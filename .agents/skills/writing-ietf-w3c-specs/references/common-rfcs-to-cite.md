# Common RFCs to Cite

Quick reference for frequently-cited RFCs in IETF submissions.

## Core Standards

### Requirements Language

```
[RFC2119]  Bradner, S., "Key words for use in RFCs to Indicate
           Requirement Levels", BCP 14, RFC 2119,
           DOI 10.17487/RFC2119, March 1997,
           <https://www.rfc-editor.org/info/rfc2119>.

[RFC8174]  Leiba, B., "Ambiguity of Uppercase vs Lowercase in
           RFC 2119 Key Words", BCP 14, RFC 8174,
           DOI 10.17487/RFC8174, May 2017,
           <https://www.rfc-editor.org/info/rfc8174>.
```

**When to cite:** Any document using MUST, SHOULD, MAY keywords.

### ABNF Grammar

```
[RFC5234]  Crocker, D., Ed. and P. Overell, "Augmented BNF for
           Syntax Specifications: ABNF", STD 68, RFC 5234,
           DOI 10.17487/RFC5234, January 2008,
           <https://www.rfc-editor.org/info/rfc5234>.

[RFC7405]  Kyzivat, P., "Case-Sensitive String Support in ABNF",
           RFC 7405, DOI 10.17487/RFC7405, December 2014,
           <https://www.rfc-editor.org/info/rfc7405>.
```

**When to cite:** Any document defining syntax with ABNF.

### IANA Considerations

```
[RFC8126]  Cotton, M., Leiba, B., and T. Narten, "Guidelines for
           Writing an IANA Considerations Section in RFCs",
           BCP 26, RFC 8126, DOI 10.17487/RFC8126, June 2017,
           <https://www.rfc-editor.org/info/rfc8126>.
```

**When to cite:** Documents creating new registries or defining registration policies.

## Normative vs Informative

### Normative References
Required to implement the specification:
- RFC 2119/8174 (if using keywords)
- RFC 5234 (if using ABNF)
- Protocol specs you depend on
- Mandatory security requirements

### Informative References
Background, context, examples:
- Historical RFCs
- Alternative approaches
- Academic papers
- External specifications not required for implementation
